# Contributions
OptiDesk is built with **security**, **seamlessness** and **speed** in mind - in that order. We take pride in our work because this software must be reliable, for hundreds more will use it, weekly, if not daily. Contributions should follow the following guidelines.

## Before starting
Please open an issue or discussion on any major changes, including:
- changes to schema
- access controls & security, and any security-sensitive behaviour
- architecture or featureset

Small bug fixes, documentation fixes and otherwise can go straight to a pull request.

## Testing your code
Run tests using `npm test`, lint using `npm run lint`.

**Do not** commit anything in gitignore, tokens, credentials, or generated local data.

## Expectations
A good PR should be detailed and explain what is changed, why/the problem being solved, include testing or be tested, and update documentation, alike README.md when required.

## Contributor license & other requirements

By submitting a contribution to OptiDesk, you agree that your contribution is licensed under AGPL-3.0-or-later, the same license as the project.

You also certify that you have the right to submit the contribution under that license.

All commits must be signed under the DCO - use the `-s` flag in your git command.
