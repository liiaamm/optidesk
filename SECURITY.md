# Security Policy

## Supported Versions
Security vulnerabilities in any version of OptiDesk that is in use in any branch of this repository are covered by this policy.

## Reporting a Vulnerability
Vulnerabilities should be immediately reported through the GitHub Private vulnerability reporting feature. If there is no response within 3 days, email [Liam at OptiDesk](mailto:liam@optidesk.dev) or the [Urgent ICT Inbox](mailto:ict@optidesk.dev).

Your message should include details about what the vulnerability is, the risk and the urgency. We aim to respond within 24-48 hours. The vulnerability will be actioned, and, where indebted, a bug bounty may be awarded at the discretion of OptiDesk.
